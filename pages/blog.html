Além do Node.js

Explorando frameworks de Javascript em diversos níveis e frentes para diferentes tipos de soluções.

Fabio Ati May

May 15 - 5 min read

Falando sobre sobre Ele: Node.js

A ~talvez~ primeira framework de backend para javascript Node.js é uma base sólida para muitas aplicações. Como o seu próprio criador quis estabelecer em sua criação:

"Eu queria ter um ambiente de programação onde eu não tivesse que me preocupar com threads e escalabilidade. [...] Node é a resposta." - Ryan Dahl.

Ele utiliza o motor V8 JavaScript da Google para interpretar o código JavaScript, oferecendo assim um ambiente de execução eficiente e de alto desempenho. Aqui estão alguns pontos-chave sobre o Node.js:

  • JavaScript no servidor: Antes do Node.js, o JavaScript era principalmente utilizado no navegador para interações do lado do cliente. Com o Node.js, o JavaScript pode ser usado no servidor, permitindo o desenvolvimento de aplicativos web completos com JavaScript em ambos os lados, cliente e servidor.
  • Assíncrono e orientado a eventos: Isso significa que, em vez de bloquear a execução enquanto espera por operações de entrada/saída (como leitura de arquivos ou consultas de banco de dados), o Node.js continua a executar outras operações. Quando a operação de entrada/saída é concluída, o Node.js retorna e executa a função de retorno de chamada associada.
  • Ampla adoção: O Node.js é amplamente adotado por empresas e desenvolvedores. Ele é usado para construir uma variedade de aplicativos, desde webapp simples até complexos, APIs, microserviços, IoT e muito mais.
  • Frameworks populares: Existem diversos frameworks populares construídos sobre o Node.js que simplificam o desenvolvimento web, como o Express.js.
  • Streaming de dados: O Node.js é eficiente no manuseio de operações que envolvem streaming de dados, como transmissão de arquivos, comunicação em tempo real e servidores de mídia.
  • NPM (Node Package Manager): O ecossistema do Node.js é enriquecido pela NPM, o gerenciador de pacotes do Node.js. A NPM é uma das maiores coleções de software livre do mundo.

Como prometido...

Segundo o Stackoverflow essas são as frameworks para javascript mais utilizadas.

  1. React.js: Desenvolvido pelo Facebook, o React é uma biblioteca JavaScript para a construção de interfaces de usuário (UI) interativas. É amplamente utilizado para o desenvolvimento de aplicativos de página única (Single Page Applications - SPAs) e é conhecido por sua eficiência e capacidade de criar componentes reutilizáveis.
  2. Angular: Desenvolvido pelo Google, o Angular é um framework JavaScript para a construção de aplicativos web de grande escala. Ele fornece uma estrutura robusta para o desenvolvimento de SPAs e oferece recursos poderosos, como injeção de dependência, vinculação de dados bidirecional e roteamento.
  3. Vue.js: O Vue.js é um framework JavaScript progressivo para a construção de interfaces de usuário. Ele é conhecido por sua simplicidade e flexibilidade, permitindo aos desenvolvedores adotarem gradualmente suas funcionalidades em seus projetos. Vue.js é frequentemente elogiado por sua curva de aprendizado suave e sua capacidade de construir aplicativos eficientes e escaláveis.
  4. Express.js: Express.js é um framework web rápido, flexível e minimalista para Node.js. Ele simplifica o desenvolvimento de aplicativos web e APIs com Node.js, fornecendo uma camada fina sobre o core do Node.js. Express.js é amplamente utilizado devido à sua simplicidade e à sua grande comunidade de desenvolvedores.
  5. jQuery: Enquanto não é estritamente uma framework, o jQuery é uma biblioteca JavaScript popular e amplamente utilizada para simplificar a manipulação do DOM, lidar com eventos, fazer requisições AJAX e realizar animações. Embora sua popularidade tenha diminuído em relação aos frameworks mais modernos, como React e Angular, o jQuery ainda é utilizado em muitos projetos web existentes.

    6. Frameworks com promessas de modernização...

Deno é uma plataforma de tempo de execução para JavaScript e TypeScript, criada por Ryan Dahl, o mesmo criador do Node.js. Deno foi projetado para superar algumas das limitações e problemas percebidos no Node.js, enquanto mantém muitas das características que o tornaram popular.

Escavando as features do Deno

  1. APIs modernas: Deno utiliza APIs modernas do navegador, o que significa que ele suporta recursos como fetch para fazer requisições HTTP, e APIs para lidar com arquivos e diretórios..
    Módulos ES nativos: Deno suporta módulos ES nativos, permitindo que você importe módulos usando a sintaxe padrão do ECMAScript sem a necessidade de usar pacotes npm ou gerenciadores de dependência externo
    Segurança por padrão: Uma das características principais do Deno é sua abordagem de segurança por padrão. Ao contrário do Node.js, onde o acesso ao sistema de arquivos, rede e outros recursos é concedido por padrão, no Deno, o acesso é negado por padrão, exigindo permissões explícitas para acessar esses recursos..Desempenho e eficiência: O Deno foi projetado para ser eficiente em termos de recursos e oferecer um desempenho sólido.

    2. Revelando a receita secreta do Bun

  2. O Bun é uma ferramenta simples e fácil de usar para criar servidores de desenvolvimento local rapidamente. Ele pode ser útil para desenvolvedores que desejam testar rapidamente aplicativos web ou sites estáticos em seu ambiente local sem a necessidade de configurar um servidor completo como o Apache ou o Nginx.Servidor HTTPS embutido: Bun suporta servidores HTTPS embutidos, o que significa que você pode testar aplicativos web em um ambiente local seguro e criptografado. Suporte a recarregamento automático: Bun suporta recarregamento automático do navegador, o que significa que você verá instantaneamente as mudanças que fizer no código refletidas no navegador sem a necessidade de atualizá-lo manualmente.
    Configuração mínima: Você não precisa configurar nada para começar a usar o Bun. Ele vem com configurações padrão que funcionam bem para a maioria dos casos de uso de desenvolvimento local. Compatibilidade com vários sistemas operacionais: Bun é compatível com Windows, macOS e Linux, tornando-o uma escolha versátil para desenvolvedores que trabalham em diferentes plataformas.
